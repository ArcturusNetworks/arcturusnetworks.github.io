To mirror the C++ counterpart, this constructor is "overloaded".

Parameters
config(dict) Configuration as json
identity(str) ID of the stream
address(str) The TCP socket the stream connects to in the form of "IP_ADDRESS:PORT"
leaky(bool)(optional) Whether the node will only use the latest frame
zeroCopy(bool)(optional)

Either use config or the other three explicit parameters to initialize the stream.

OutgoingStream(config=json_dict)
# or
OutgoingStream(
identity='stream-name',
address='127.0.0.1:5000',
leaky=False)

this function is indeed the cause of the problem.

This loop never appears to run so as a result, streamproc::IncomingStream::ProcessExternal sends a "Request" message but never receives anything back, thereby making its internal socket buffer empty, thus blocks its own Process, which makes the thread time out and we get to observe those timeout messages. (this is solved by actually starting the thread :O)
