Detailed Description

The Python equivalent of streamproc::IncomingStream.

Very similar to pystreamproc.stream.OutgoingStream in terms of both code structure and variable usage; except that now the ProcessExternal receives messages instead of sending them.

Parameters
config(dict) Configuration as json
identity(str) ID of the stream
address(str) The TCP socket the stream connects to in the form of "IP_ADDRESS:PORT"
leaky(bool) Whether the node will only use the latest frame

Either use config or the other three explicit parameters to initialize the stream like

IncomingStream(config=json_dict)
# or
IncomingStream(
identity='stream-name',
address='127.0.0.1:5000',
leaky=True)

def pystreamproc.stream.IncomingStream.ProcessInternal (   self)

From here the Python implementation diverges from the C++ paradigm because pass by reference can only be done in Python when mutating the members of the arguments, thus making this unpredictable so it's much more natural to return than to mutate the referenced argument.

Hence there's no need for the more generic streamproc::Stream::Process and ProcessInternal will be merged with process.

process will get a zmq bytestring from the external socket buffer then decode it into an image frame (numpy.ndarray).
