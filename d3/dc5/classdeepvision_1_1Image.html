Detailed Description

A container for image data obtained via OpenCV, with methods for normalization and quantization.

For float input models, pixel channel values normally need to be mapped to [-1, 1]. This can be done by dividing by 127.5, then adding -1. Other values for scale/offset may be passed in to Normalize. See https://github.com/tensorflow/models/tree/master/research/object_detection/models for the normalization requirements of different Tensorflow models.

For quantized models, image bytes are converted to normalized (1/255) floats. Then these are passed to QuantizeVector, which correctly converts them back to uint8_t using a Tensorflow-like quantization specification. See third_party/armnn_tests/include/QuantizeHelper.hpp

Requires:

  • Input cv::Mat or image file are non-empty, pixel channels are 8-bit 3-channel

Guarantees:

  • Once constructed, the underlying byte array is non-empty, pixel channels are uint8_t RGB
  • Row-oriented memory layout
