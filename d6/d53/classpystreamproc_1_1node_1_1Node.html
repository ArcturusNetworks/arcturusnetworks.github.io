Detailed Description

Python-native ABC equivalent of streamproc::Node.

Using an Abstract Base Class requires all functions to be defined in its children to be instantiated, just like virtual functions in its C++ mirror.

About logging: Using Python logging.Logger so to log your own messages, just

self.logger.info('non-frequent information')
self.logger.warn('default min level')
self.logger.exception('also prints the stack trace with it')
self.logger.debug('the lowest "level"')

You may specify the minimum logging level in the constructor

formatter = logging.Formatter(
'[%(asctime)s] [DERIVED_CLASS] [%(levelname)s] %(message)s'
)
self._handler.setFormatter(formatter)
# optionally, set the minimum logging level
self.logger.setLevel(logging.DEBUG)

in the inherited class constructor. Too bad this means that even messages in the inherited functions will show up as though it came from the child class(es). Don't forget to call super().__init__ of course. This applies to additional Streams as well.
