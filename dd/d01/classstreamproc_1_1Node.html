Detailed Description

Base class for node.

A computer vision pipeline is composed of nodes which are responsible for executing a distinct task. This class manages a worker routine where a node performs a single task repeatedly.

In order to perform its task, a node must process some data which it may send and/or receive to other nodes in a network. A node has two main pathways for communication: 

Streams: transmit data which is to be transformed by the pipeline in some meaninful way
Bus: transmit signals, state transitions, statistics and other metadata about the pipeline

In order to implement a custom node class, do not derive from this class. Instead derive from either the Source, Filter, or Sink classes defined below. These three classes are important for defining explicit, simple, and meaningful roles for nodes within a pipeline.

See also
streamproc::Source, streamproc::Filter, streamproc::Sink
